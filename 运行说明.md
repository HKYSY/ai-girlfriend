# 虚拟伴侣 - 运行说明

本项目有两种使用方式：
- **方式一：开发模式**（前后端分离，适合开发和调试）
- **方式二：桌面应用**（打包成 exe，双击即可运行，适合日常使用）

## 一、安装 Node.js（必做，只需一次）

1. 打开 https://nodejs.org
2. 下载 **LTS 版本**（长期支持版，更稳定）
3. 双击安装，一路点"下一步"即可
4. 安装完成后，打开终端（Win+R 输入 cmd 回车），输入：
   ```
   node -v
   ```
   如果显示版本号（如 v20.x.x），说明安装成功

---

## 二、获取 DeepSeek API Key（必做，AI 聊天需要）

1. 打开 https://platform.deepseek.com
2. 注册账号并登录
3. 点左侧 **API Keys** → **创建 API Key**
4. 复制生成的 Key（格式：`sk-xxxxxxxx`）

---

## 三、配置后端

1. 把 `ai-girlfriend.zip` 解压到一个文件夹
2. 进入 `backend` 文件夹
3. 找到 `.env.example` 文件，复制一份
4. 把复制的那份重命名为 `.env`（注意：没有后缀名）
5. 用记事本打开 `.env`，修改为：
   ```
   DEEPSEEK_API_KEY=你刚才复制的Key
   DEEPSEEK_MODEL=deepseek-v4-flash
   PORT=3001
   ```
6. 保存文件

---

## 四、启动后端

1. 在 `backend` 文件夹里打开终端
   - 方法：在文件夹地址栏输入 `cmd` 回车
2. 输入以下命令安装依赖（只需第一次运行）：
   ```
   npm install
   ```
3. 等待安装完成后，编译项目：
   ```
   npm run build
   ```
4. 启动后端：
   ```
   npm start
   ```
5. 看到 `服务器运行在端口 3001` 表示成功
6. **这个终端窗口不要关**，保持运行

---

## 五、启动前端

1. 另开一个终端窗口（在 `frontend` 文件夹里打开）
   - 方法：进入 frontend 文件夹，在地址栏输入 `cmd` 回车
2. 安装依赖（只需第一次运行）：
   ```
   npm install
   ```
3. 启动前端：
   ```
   npm run dev
   ```
4. 终端会显示一个网址，类似：
   ```
   Local: http://localhost:5173/
   ```
5. 用浏览器打开那个网址就能用了

---

## 常见问题

### Q: npm install 很慢怎么办？
A: 可以换国内镜像源，在终端运行：
```
npm config set registry https://registry.npmmirror.com
```
然后重新运行 `npm install`

### Q: 后端启动报错 "未配置 API Key"？
A: 检查 `backend/.env` 文件是否正确填写了 DEEPSEEK_API_KEY

### Q: 前端打开后没有角色？
A: 页面会自动创建一个默认角色"玉子"，刷新页面即可

### Q: 关闭后重新启动怎么办？
A: 后端和前端各开一个终端，分别运行 `npm start`（后端）和 `npm run dev`（前端）即可，不需要重新 install

### Q: 两个终端是什么意思？
A: 后端和前端是两个独立程序，需要同时运行。开两个终端窗口，一个运行后端，一个运行前端，两个都不能关。

---

## 六、桌面应用打包（Electron）

如果不想每次都开两个终端，可以打包成 Windows 桌面应用，双击 exe 即可运行。

### 前置条件
- 已完成上面的"安装 Node.js"和"获取 DeepSeek API Key"
- 已配置好 `backend/.env`

### 打包步骤

1. 在项目**根目录**（不是 backend 或 frontend）打开终端
2. 安装根目录依赖（只需第一次）：
   ```
   npm install
   ```
3. 打包桌面应用：
   ```
   npm run electron:build
   ```
4. 等待打包完成（约 2-3 分钟），完成后在 `out\win-unpacked\` 目录里找到 `虚拟伴侣.exe`
5. 双击 `虚拟伴侣.exe` 即可运行

### 打包安装包（给别人用）

```
npm run electron:dist
```
完成后在 `out\` 目录里会生成 `虚拟伴侣 Setup.exe` 安装包，发给别人安装即可。

### 桌面版说明
- 数据存储在 `%AppData%\虚拟伴侣\` 目录，与开发模式数据隔离
- 启动日志在 `%AppData%\virtual-companion\main.log`，出问题可以查看
- 如果端口 3001 被占用（比如开发模式后端在跑），桌面版会自动复用已有服务，不报错
- 自定义图标放在 `build\icon.ico`

### 更新代码后重新打包

改了前端或后端代码后，重新运行 `npm run electron:build` 即可，桌面版会同步最新代码。数据不会丢失。
