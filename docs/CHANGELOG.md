## [v1.7] - 2026-07-24

### ✨ 新功能

#### 代码重构
- **前端 Hook 拆分**: 将 App.tsx 重构为模块化架构
  - 提取 `useChat` Hook: 管理聊天消息、发送逻辑、主动消息
  - 提取 `useMood` Hook: 管理心情值、情感状态、心情联动色调
  - App.tsx 代码量减少 24% (从 1088 行 → 826 行)
- **API 错误处理优化**: 增强前端稳定性
  - 实现请求重试机制(最多 3 次,指数退避)
  - 添加友好错误提示(技术错误转用户可理解提示)
  - 10+ 个关键 API 函数添加重试逻辑
- **后端日志系统**: 添加结构化日志工具
  - 实现统一的 JSON 格式日志输出
  - 支持 info/warn/error/debug 四种日志级别
  - 关键位置替换 console.log 为 logger

#### 项目清理
- 删除垃圾文件: 日志文件、临时配置、构建缓存(共 10 个)
- 删除垃圾目录: `.impeccable/`、`skills/`(共 2 个)
- 删除死代码: 未使用的导入、被注释的代码块(共 57 行)
- 代码质量提升: 项目更整洁,无冗余代码

#### 文档更新
- 新增 `CONTRIBUTING.md`: 贡献指南、开发环境搭建、代码规范
- 新增 `docs/API.md`: 后端 API 文档(40+ 端点)
- 新增 `docs/ARCHITECTURE.md`: 项目架构说明、数据流图
- 更新 `PRODUCT.md`: 补充 v1.7 功能清单和未来规划
- 更新 `DESIGN.md`: 补充阶段 3-4 设计实现说明

#### UI 重构
- **心情联动色调系统**: 界面极光色调根据 AI 心情值动态变化
  - 心情值 90-100：偏粉色（`oklch(0.75 0.18 340)`）
  - 心情值 60-89：偏紫色（`oklch(0.65 0.15 300)`）
  - 心心情值 0-59：偏冷蓝（`oklch(0.45 0.08 260)`）
- **时间同步机制**: AI 感知具体日期和时间，回复更自然
  - 每次聊天注入当前日期信息："今天是 2026年7月24日（周四）"
  - 检测距上次聊天的时间差，AI 会自然地表现出想念或抱怨
- **每日问候优化**: 每天首次打开时 75% 概率触发 AI 主动问候
  - 严格限制问候类型：询问去处、表达关心、表达想念
  - 避免自言自语、讲故事等无关内容

#### 时间感知系统
- 检测用户长时间不活跃（≥1 天），AI 自动调整心情
  - 1 天未活跃：心情 -30
  - 2 天未活跃：心情 -50
  - 3 天及以上：心情归零
- AI 在回复中自然体现对时间流逝的反应（生气、委屈、想念等）

### 🔧 优化

- **对话摘要系统**: 修复摘要生成逻辑，防止新旧摘要合并丢失信息
- **日记生成字数**: 增加日记篇幅（80-600 字），让内容更充实
- **AI 回复场景描写检测**: 新增第二重保护，检测并警告 AI 回复中的场景描写关键词
- **心情变化保底机制**: AI 输出情绪标签但心情无明显变化时强制调整（生气时心情必降，开心时必升）

### 🐛 Bug 修复

- 修复 AI 回复偶尔包含场景描写的问题（如"愣了一下"、"眼神..."等）
- 修复长时间不活跃后心情惩罚未生效的问题
- 修复每日问候重复触发的问题

## [v1.6] - 2026-06-15

### ✨ 新功能

#### 表情包系统重构
- **批量扫描导入**: 一键扫描 `data/stickers/temp/` 目录，批量导入用户放入的图片素材
- **毛玻璃 UI**: 表情包面板采用现代毛玻璃效果设计
- **最近使用**: 新增"最近"标签页，快速访问常用表情
- **批量删除**: 勾选多个表情包一键删除
- **GIF 首帧冻结**: 避免 GIF 动态图在面板中闪烁

#### 表情包发送流程
- 用户发送表情包触发 AI 流式回复（SSE）
- AI 可根据情绪自动匹配表情包回复（概率控制）
- 概率动态调整：2 分钟内发过 → 10%，5 分钟内 → 20%

### 🔧 优化

- **后端路由拆分**: 将 `index.ts` 中的路由按功能拆分到 `routes/` 目录
  - `character.ts`: 角色管理
  - `diary.ts`: 日记查询
  - `mood.ts`: 心情历史
  - `sticker.ts`: 表情包管理
  - `upload.ts`: 文件上传
- **数据库查询优化**: 简化表情包查询逻辑，避免复杂 SQL 卡住

### 🐛 Bug 修复

- 修复表情包临时目录被静态服务暴露的安全问题
- 修复批量导入时文件名冲突导致覆盖的问题
- 修复表情包使用次数统计不准确的问题

## [v1.5] - 2026-05-20

### ✨ 新功能

#### Live2D 集成
- **Cubism 4 模型渲染**: 使用 `pixi-live2d-display-mulmotion` 库
- **交互功能**:
  - 鼠标拖拽、滚轮缩放
  - 视线跟随鼠标
  - 点击触发动作
  - 表情切换（8 种情绪）
- **预设模型**: 内置 IceGirl 和 Haru 两款模型

#### 基础聊天功能
- **SSE 流式输出**: 逐字显示，像真人发微信
- **10 级心情系统**: 心情值 0-100，影响回复语气
- **8 种情绪标签**: 开心、生气、难过、撒娇、惊讶、疑惑、害羞、平静
- **情感分析兜底**: AI 未输出心情标记时自动分析用户消息情感

#### 表情包功能
- 用户上传表情包（支持 JPG/PNG/GIF/WEBP）
- 表情包分类管理（happy/angry/cute/confused/sad/general）
- 情绪标签标注（匹配 AI 回复情绪）

### 🔧 优化

- **Token 优化**: 对话分级发送，近 10 条原文 → 11-25 条截断 → 26+ 走摘要
- **对话永久保留**: SQLite 数据库存储，不自动删除
- **智能搜索**: 按关键词搜索历史聊天

## [v1.1] - 2026-01-15

- 从 JSON 文件存储迁移到 SQLite
